# 家人健康监测 - 微信小程序 PRD

## 一、项目概述

| 项目 | 说明 |
|------|------|
| 产品名称 | 家人健康监测（暂定） |
| 目标用户 | Marvel 的父母（65岁），后续可开放给其他家庭 |
| 核心价值 | 异地子女远程掌握父母健康状况，激励父母坚持运动 |
| 平台 | 微信小程序 |
| 提醒方式 | 家庭微信群人工提醒（不做小程序推送） |

## 二、用户角色

| 角色 | 说明 |
|------|------|
| 家长用户 | Marvel 的父母，日常使用者，打卡 + 上传体检报告 |
| 子女用户 | Marvel，查看父母数据，参与 PK |
| 管理员 | Marvel，系统管理 |

> 初期不区分角色权限，所有注册用户功能一致。

## 三、功能清单

### P0 - 核心功能（MVP）

#### 3.1 微信登录
- 微信一键授权登录，无需手机号注册
- 首次登录填写：昵称、年龄、体重（用于卡路里计算）
- 体重支持后续修改

#### 3.2 体检报告 AI 分析
- 用户拍照或从相册上传体检报告图片（支持多张）
- 后端 OCR 识别报告文字内容
- 调用 LLM（Claude API）分析报告，输出：
  - 检查项目摘要（用通俗语言解释每项指标）
  - 异常指标标红高亮
  - 健康建议（饮食、运动、复查周期）
  - 需要关注/就医的情况标注
- 每份报告存档，可在「我的报告」列表中查看历史
- **底部固定免责声明**：「以上分析由 AI 生成，仅供参考，不构成医疗诊断。如有不适请及时就医。」

#### 3.3 每日运动打卡
- 用户选择运动类型（预设列表）：
  - 散步、快走、慢跑
  - 太极拳、广场舞
  - 骑自行车
  - 家务劳动
  - 康复训练（针对膝关节等）
  - 自定义输入
- 填写运动时长（分钟）
- 可选：拍照/短视频作为打卡凭证（上传云端存档）
- 系统根据「运动类型 MET 值 x 体重 x 时长」自动计算消耗卡路里
- 打卡成功展示：今日消耗卡路里 + 连续打卡天数 + 鼓励语
- 每天可多次打卡（累加计算）

**MET 值参考表（代谢当量）：**

| 运动类型 | MET 值 |
|----------|--------|
| 散步（慢） | 2.5 |
| 快走 | 3.5 |
| 慢跑 | 7.0 |
| 太极拳 | 3.0 |
| 广场舞 | 4.5 |
| 骑自行车 | 4.0 |
| 家务劳动 | 3.0 |
| 康复训练 | 2.5 |

> 卡路里计算公式：消耗(kcal) = MET x 体重(kg) x 时长(h)

#### 3.4 PK 功能
- 用户之间可发起 PK 挑战（分享小程序码邀请）
- PK 维度：
  - 周 PK：本周（周一至周日）累计卡路里
  - 月 PK：本月累计卡路里
- 卡路里转换为「健康指数」：健康指数 = 累计卡路里 / 100（取整）
- PK 页面展示：
  - 双方头像、昵称
  - 当前健康指数对比（进度条）
  - 今日/本周/本月数据
  - 领先/落后状态提示
- PK 结束后展示结果（可分享到微信聊天）

### P1 - 增强功能（二期）

#### 3.5 数据看板（子女视角）
- 查看绑定家人的打卡记录和运动趋势
- 周/月运动数据折线图
- 未打卡提醒（在看板上标红）

#### 3.6 康复训练计划
- 针对母亲的膝关节问题，提供专项训练建议：
  - 直腿抬高（股四头肌强化）
  - 靠墙静蹲（膝关节稳定性）
  - 坐姿伸膝（关节活动度）
- 每个动作配图文/视频说明
- 可标记完成，计入打卡

#### 3.7 健康知识推送
- 定期更新适合老年人的健康科普文章
- 根据已上传的体检报告，推荐相关内容

### P2 - 远期功能

- 接入智能手环/手表数据（步数、心率）
- 家庭成员群组管理
- 用药提醒
- 复查日历提醒

## 四、页面结构

```
首页（Tab 1）
├── 今日打卡卡片（快捷入口）
├── 连续打卡天数
├── 本周运动概览（卡路里 + 健康指数）
└── 进行中的 PK 状态

运动打卡（Tab 2）
├── 选择运动类型
├── 输入时长
├── 拍照/视频（可选）
├── 提交打卡
└── 打卡历史列表

PK 排行（Tab 3）
├── 当前 PK 列表
├── 发起/接受 PK
├── 周榜 / 月榜
└── PK 详情页

我的（Tab 4）
├── 个人信息（昵称、体重等）
├── 我的体检报告
│   ├── 上传报告
│   └── 历史报告列表 → 报告详情
├── 打卡日历（月视图）
├── 运动统计
└── 设置
```

## 五、技术方案

### 前端
- **框架**：微信小程序原生 或 uni-app（跨端，方便后续出 App）
- **UI 组件**：WeUI / Vant Weapp
- **图表**：wx-charts 或 ECharts 微信版

### 后端
- **运行时**：Node.js（推荐）或 Python
- **框架**：Express / Koa 或 FastAPI
- **数据库**：MySQL 或 PostgreSQL（用户、打卡、PK 数据）
- **对象存储**：腾讯云 COS（图片/视频存储）
- **OCR**：腾讯云 OCR 通用印刷体识别（体检报告）
- **AI 分析**：Claude API（分析体检报告、生成健康建议）
- **部署**：腾讯云（与微信生态契合度最高）

### 简化方案（推荐初期）
- **微信云开发**：免服务器，自带数据库 + 存储 + 云函数
- 省去运维成本，适合个人开发者快速上线
- 后续用户量大了再迁移到独立服务器

- [ ] **下载微信开发者工具**：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
- [ ] **开通云开发**：在开发者工具中开通（免费额度够用）
- [ ] **申请腾讯云 OCR**：https://cloud.tencent.com/product/ocr （有免费额度）
- [ ] **准备 Claude API Key**：用于体检报告分析（你应该已有）

### 产品相关
- [ ] 确定小程序名称（需在微信后台审核通过）
- [ ] 准备小程序头像/Logo
- [ ] 确认母亲的体重（用于默认卡路里计算）
- [ ] 收集一份母亲的体检报告样本（用于测试 OCR + AI 分析效果）

### 开发阶段（我来主导，你配合）
- [ ] 每个功能我写好代码后，你在开发者工具中预览测试
